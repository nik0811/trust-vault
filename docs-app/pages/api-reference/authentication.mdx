# Authentication

SecureLens supports multiple authentication methods for API access.

## API Key Authentication

API keys are the recommended method for server-to-server integrations.

### Getting Your API Key

1. Log in to SecureLens
2. Go to **Settings** → **API Keys**
3. Click **Generate Key**
4. Give it a name and select permissions
5. Copy the key (it won't be shown again)

### Using Your API Key

Include the API key in the `X-API-Key` header:

```bash
curl -X GET "https://api.securelens.ai/v1/data-sources" \
  -H "X-API-Key: your-api-key"
```

Or as a query parameter (not recommended for production):

```bash
curl "https://api.securelens.ai/v1/data-sources?api_key=your-api-key"
```

### API Key Permissions

When creating an API key, select the appropriate scope:

| Scope | Permissions |
|-------|-------------|
| `read` | Read-only access |
| `write` | Create and update resources |
| `delete` | Delete resources |
| `admin` | Full access |

## JWT Authentication

For user-based authentication (e.g., custom integrations):

### Login

```bash
POST https://api.securelens.ai/v1/auth/login
Content-Type: application/json

{
  "email": "user@company.com",
  "password": "your-password"
}
```

Response:

```json
{
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIs...",
    "refresh_token": "eyJhbGciOiJIUzI1NiIs...",
    "expires_in": 3600,
    "token_type": "Bearer"
  }
}
```

### Using the Token

Include the token in the Authorization header:

```bash
curl -X GET "https://api.securelens.ai/v1/data-sources" \
  -H "Authorization: Bearer your-access-token"
```

### Refreshing Tokens

When the access token expires, use the refresh token:

```bash
POST https://api.securelens.ai/v1/auth/refresh
Content-Type: application/json

{
  "refresh_token": "your-refresh-token"
}
```

## OAuth 2.0 / SSO

For enterprise SSO integration, SecureLens supports OAuth 2.0 and OIDC.

Contact your account manager to configure SSO for API access.

## Security Best Practices

1. **Never expose API keys** in client-side code or public repositories
2. **Use environment variables** to store credentials
3. **Rotate API keys** regularly (recommended: every 90 days)
4. **Use scoped API keys** with minimal required permissions
5. **Monitor API key usage** in the audit trail

## Revoking API Keys

If an API key is compromised:

1. Go to **Settings** → **API Keys**
2. Find the compromised key
3. Click **Revoke**
4. Generate a new key
5. Update your applications
