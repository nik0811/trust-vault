### Steps

1. **Extract** вҖ” Scanner connects to data source, extracts sample data
2. **Classify** вҖ” GLiNER model analyzes text for PII
3. **Store** вҖ” Results saved to PostgreSQL
4. **Publish** вҖ” Event published to Kafka
5. **Handle** вҖ” Event handlers process (audit, notify, etc.)

## AI Gate Flow

```
в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ     в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ     в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ
в”Ӯ    User     в”Ӯв”Җв”Җв”Җв”Җв–¶в”Ӯ   AI Gate   в”Ӯв”Җв”Җв”Җв”Җв–¶в”Ӯ     LLM     в”Ӯ
```

### Steps

1. **Receive** вҖ” User sends prompt to AI Gate
2. **Scan** вҖ” Classify prompt for PII
3. **Redact** вҖ” Mask sensitive data
4. **Forward** вҖ” Send clean prompt to LLM
5. **Filter** вҖ” Scan response for leakage
6. **Return** вҖ” Return filtered response
7. **Audit** вҖ” Log entire interaction

## Policy Evaluation Flow

```
в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ     в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ     в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ
в”Ӯ   Request   в”Ӯв”Җв”Җв”Җв”Җв–¶в”Ӯ   Policy    в”Ӯв”Җв”Җв”Җв”Җв–¶в”Ӯ   Action    в”Ӯ
в”Ӯ  (Access)   в”Ӯ     в”Ӯ   Engine    в”Ӯ     в”Ӯ (Allow/Deny)в”Ӯ

                    в”Ӯ   Match     в”Ӯ
                    в”Ӯ  Policies   в”Ӯ
                    в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

### Steps

1. **Request** вҖ” Data access request received
2. **Context** вҖ” Gather user, resource, classification info
3. **Match** вҖ” Find applicable policies
4. **Evaluate** вҖ” Check conditions
5. **Decide** вҖ” Determine action (allow/deny/redact)
6. **Execute** вҖ” Apply action
7. **Log** вҖ” Audit the decision

## Event Processing

### Event Types

| Event | Trigger | Handlers |
|-------|---------|----------|
| `classification.created` | New classification | Audit, Label, Quality |
| `policy.violated` | Policy breach | Alert, Ticket, Audit |
| `scan.completed` | Scan finished | Report, Notify |
| `dsar.received` | Privacy request | Workflow, Notify |

### Event Schema

```json
{
  "event_id": "evt-123",
  "event_type": "classification.created",
  "timestamp": "2024-01-20T10:30:00Z",
  "tenant_id": "tenant-abc",
  "data": {
    "classification_id": "cls-456",
    "data_source_id": "ds-789",
    "type": "EMAIL",
    "confidence": 0.98
  }
}
```

## Data Lineage

SecureLens tracks data lineage using OpenLineage:

```
Source вҶ’ Transform вҶ’ Target
   в”Ӯ         в”Ӯ          в”Ӯ
   в–ј         в–ј          в–ј
DataHub  DataHub    DataHub
(input)  (job)     (output)
```

### Lineage Events

```json
{
  "eventType": "COMPLETE",
  "job": {
    "namespace": "securelens",
    "name": "classification-scan"
  },
  "inputs": [{
    "namespace": "prod-db",
    "name": "users"
  }],
  "outputs": [{
    "namespace": "securelens",

  }]
}
```
