## Design Principles

### 1. Clean Architecture

SecureLens follows Clean Architecture principles:

- **Domain Layer** — Business rules and entities
- **Application Layer** — Use cases and orchestration
- **Interface Layer** — API handlers and adapters
- **Infrastructure Layer** — Database, Kafka, external services

### 2. Event-Driven

Side effects are handled through events:

```
Action → Event Published → Event Handlers
                              │
                    ┌─────────┼─────────┐
                    ▼         ▼         ▼
                 Audit    Notify    Update
                  Log     Slack     Cache
```

### 3. Multi-Tenant by Design

Every operation is tenant-scoped:

- `tenant_id` on every database table
- Tenant context in every request
- Isolated data access
- Per-tenant configuration

### 4. Horizontal Scalability

Components scale independently:

- Stateless API servers
- Kafka consumer groups
- Database read replicas
- Vector DB sharding

## Technology Stack

| Layer | Technology | Purpose |
|-------|------------|---------|
| **Frontend** | React, Next.js | Web UI |
| **API** | Go, Chi router | REST API |
| **Database** | PostgreSQL | Primary data store |
| **Events** | Kafka (KRaft) | Event streaming |
| **Vector DB** | Qdrant | Embeddings storage |
| **Classification** | GLiNER (ONNX) | PII detection |
| **OCR** | PaddleOCR | Document processing |
| **Metadata** | DataHub | Data catalog |

## Deployment Options

### Docker Compose (Development)

```yaml
services:
  api:
    image: securelens/api:latest
    ports: ["8080:8080"]
  web:
    image: securelens/web:latest
    ports: ["3000:3000"]
  postgres:
    image: postgres:15
  kafka:
    image: bitnami/kafka:latest
  qdrant:
    image: qdrant/qdrant:latest
```

### Kubernetes (Production)

```yaml
# Helm chart available
helm install securelens securelens/securelens \
  --set replicas.api=3 \
  --set replicas.worker=5
```

### Cloud-Native

- **AWS**: EKS, RDS, MSK, S3
- **Azure**: AKS, Azure DB, Event Hubs
- **GCP**: GKE, Cloud SQL, Pub/Sub
