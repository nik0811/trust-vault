# Reports

Generate compliance, operational, and executive reports.

## Overview

SecureLens provides comprehensive reporting:

- **Compliance reports** for auditors
- **Executive dashboards** for leadership
- **Operational reports** for teams

## Generating Reports

### On-Demand Reports

1. Navigate to **Reports**
2. Select report type
3. Configure parameters:

```yaml
report:
  type: COMPLIANCE_SUMMARY
  framework: GDPR
  period:
    start: 2024-01-01
    end: 2024-01-31
  scope:
    tenants: [all]
    data_sources: [all]
  format: PDF
  include:
    - executive_summary
    - gap_analysis
    - evidence
    - recommendations
```

4. Click **Generate**

### Scheduled Reports

```yaml
scheduled_report:
  name: Weekly Compliance Report
  type: COMPLIANCE_SUMMARY
  schedule: "0 8 * * MON"  # Monday 8 AM
  config:
    framework: [GDPR, CCPA]
    format: PDF
  delivery:
    email:
      to: [compliance@company.com, ciso@company.com]
      subject: "Weekly Compliance Report - {{date}}"
    slack:
      channel: "#compliance-reports"
```

## Report Delivery

### Delivery Options

| Method | Configuration |
|--------|---------------|
| **Email** | Recipients, subject, body |
| **Slack** | Channel, message |
| **S3** | Bucket, path |
| **SFTP** | Host, credentials |
| **Webhook** | URL, headers |

### Delivery Configuration

```yaml
delivery:
  email:
    to: [compliance@company.com]
    cc: [ciso@company.com]
    subject: "{{report_name}} - {{date}}"
    body: |
      Please find attached the {{report_name}}.
      
      Summary:
      - Compliance Score: {{compliance_score}}%
      - Open Issues: {{open_issues}}
      
    attachments: [report.pdf]
    
  s3:
    bucket: compliance-reports
    path: "{{year}}/{{month}}/{{report_name}}.pdf"
    
  slack:
    channel: "#compliance"
    message: "New report available: {{report_name}}"
    upload: true
```

## Custom Reports

### Report Builder

Create custom reports with the visual builder:

```yaml
custom_report:
  name: Monthly PII Summary
  sections:
    - type: METRIC_CARDS
      metrics:
        - total_records
        - pii_records
        - compliance_score
        
    - type: CHART
      chart_type: BAR
      data: pii_by_type
      title: "PII by Category"
      
    - type: TABLE
      data: top_pii_sources
      columns: [source, pii_count, percentage]
      title: "Top PII Sources"
      
    - type: TEXT
      content: |
        ## Recommendations
        Based on this month's analysis...
```

### SQL Reports

Write custom SQL queries:

```sql
-- Custom PII exposure report
SELECT 
  ds.name as data_source,
  c.classification_type,
  COUNT(*) as record_count,
  AVG(c.confidence) as avg_confidence
FROM classifications c
JOIN data_sources ds ON c.data_source_id = ds.id
WHERE c.created_at >= '{{start_date}}'
  AND c.created_at < '{{end_date}}'
GROUP BY ds.name, c.classification_type
ORDER BY record_count DESC
```

## API Access

### Generate Report

```bash
curl -X POST "https://api.securelens.ai/v1/reports/generate" \
  -H "Authorization: Bearer $API_KEY" \
  -d '{
    "type": "COMPLIANCE_SUMMARY",
    "framework": "GDPR",
    "format": "PDF"
  }'
```

### Get Report Status

```bash
curl -X GET "https://api.securelens.ai/v1/reports/rpt-123/status" \
  -H "Authorization: Bearer $API_KEY"
```

### Download Report

```bash
curl -X GET "https://api.securelens.ai/v1/reports/rpt-123/download" \
  -H "Authorization: Bearer $API_KEY" \
  -o report.pdf
```
