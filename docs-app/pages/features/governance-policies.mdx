# Governance Policies

Define and enforce data governance rules across your organization.

## What Policies Do

Governance policies automate data protection by defining rules for:

- **Access Control** — Who can access what data
- **Redaction** — Automatically mask sensitive data
- **Retention** — Data lifecycle management
- **Residency** — Geographic restrictions
- **AI Governance** — LLM usage rules

## Policy Types

### Access Policies

Control who can access sensitive data based on their role:

**Example:** Only compliance and admin roles can access highly confidential data.

### Redaction Policies

Automatically mask sensitive data in exports, API responses, or AI prompts:

**Example:** SSN values are displayed as `XXX-XX-1234` (showing only last 4 digits).

### Retention Policies

Enforce data lifecycle rules:

**Example:** Customer data is archived after 7 years and deleted after 10 years.

### Residency Policies

Restrict where data can be stored or transferred:

**Example:** EU personal data cannot be transferred to US regions.

### AI Policies

Govern how data flows to AI/LLM systems:

**Example:** SSN and credit card numbers are redacted before sending to any LLM.

## Creating a Policy

### Step 1: Navigate to Policies

1. Go to **Governance** → **Policies**
2. Click **Create Policy**

### Step 2: Choose Policy Type

Select the type of policy you want to create:
- Access
- Redaction
- Retention
- Residency
- AI Governance

### Step 3: Define Conditions

Specify when this policy applies:

| Condition | Description |
|-----------|-------------|
| **Data Sources** | Which sources this applies to |
| **PII Types** | Which data types (e.g., SSN, EMAIL) |
| **Sensitivity Labels** | Which sensitivity levels |
| **Regions** | Geographic locations |

### Step 4: Configure Rules

Add rules with actions:

| Action | What It Does |
|--------|--------------|
| **Allow** | Permit access |
| **Deny** | Block access |
| **Redact** | Mask the data |
| **Audit** | Log the access |
| **Alert** | Send notification |
| **Require Approval** | Trigger workflow |

### Step 5: Set Enforcement Mode

Choose how the policy is enforced:

| Mode | Behavior |
|------|----------|
| **Monitor** | Log violations but don't block |
| **Warn** | Alert user but allow override |
| **Enforce** | Block violations |

### Step 6: Activate

1. Review the policy summary
2. Click **Activate**
3. The policy takes effect immediately

## Managing Policies

### View Policy Details

1. Click on a policy name
2. See conditions, rules, and recent activity
3. View violation history

### Edit a Policy

1. Click on a policy
2. Click **Edit**
3. Make changes
4. Click **Save**

Changes create a new version. Previous versions are retained for audit purposes.

### Test Before Activating

Test policies before they go live:

1. Click **Test Policy**
2. Select test data
3. View simulated results
4. Adjust if needed

### Disable or Delete

- **Disable** — Temporarily stop enforcement
- **Delete** — Remove the policy entirely

## Monitoring Policies

### Policy Dashboard

View policy effectiveness:

| Metric | What It Shows |
|--------|---------------|
| **Evaluations** | Policy checks performed |
| **Violations** | Rules triggered |
| **Blocks** | Access denied |
| **Overrides** | User overrides (if allowed) |

### Violation Alerts

Get notified when policies are violated:

1. Go to **Settings** → **Notifications**
2. Enable policy violation alerts
3. Choose notification method (email, Slack, etc.)

## Common Use Cases

### Protect PII from Unauthorized Access

Create an access policy that:
- Applies to all data with PII classifications
- Allows access only to authorized roles
- Logs all access attempts

### Redact SSN in Exports

Create a redaction policy that:
- Applies to SSN data type
- Masks values in exports and API responses
- Shows only last 4 digits

### Prevent PII in AI Prompts

Create an AI policy that:
- Scans all prompts sent through AI Gate
- Redacts SSN, credit cards, and health data
- Allows names and emails with audit logging
