# Audit Trail

Complete logging of all data access and governance actions.

![Audit Trail](/screenshots/audit-trail.png)

## What the Audit Trail Captures

SecureLens maintains a comprehensive audit trail of:

- **Data Access** — Who accessed what data, when
- **Policy Actions** — Governance decisions made
- **Classification Changes** — Labels assigned or modified
- **AI Gate Activity** — All LLM interactions
- **User Actions** — Logins, configuration changes

## Viewing Audit Logs

### Navigate to Audit Trail

1. Click **Audit Trail** in the sidebar
2. View the list of recent events
3. Use filters to find specific events

### Filter Options

| Filter | Description |
|--------|-------------|
| **Time Range** | Last hour, day, week, or custom |
| **Category** | Data access, policy, user, etc. |
| **User** | Specific user or service |
| **Resource** | Data source, table, column |
| **Result** | Allowed, denied, error |

### Search Examples

Find all PII access in the last week:
- Category: Data Access
- Time: Last 7 days
- Filter: PII types

Find policy violations:
- Category: Policy
- Result: Denied

## Event Categories

| Category | What It Logs |
|----------|--------------|
| **Data Access** | Queries, exports, views |
| **Classification** | Labels assigned, changed |
| **Policy** | Evaluations, violations |
| **AI Gate** | Prompts, responses, redactions |
| **User** | Login, logout, permission changes |
| **System** | Config changes, scans completed |

## Event Details

Each audit event includes:

- **Timestamp** — When it happened
- **User** — Who performed the action
- **Action** — What was done
- **Resource** — What was accessed
- **Result** — Success, denied, or error
- **Context** — Additional details

## Data Access Auditing

Every data query is logged with:

| Field | Description |
|-------|-------------|
| **User** | Who ran the query |
| **Source** | Data source accessed |
| **Tables** | Tables queried |
| **Columns** | Columns accessed |
| **Rows** | Records returned |
| **Duration** | Query time |

For sensitive data, additional details include:
- PII types accessed
- Sensitivity labels
- Policy evaluation results
- Any redaction applied

## Exporting Audit Logs

Export logs for compliance or analysis:

### Export Formats

- **CSV** — For spreadsheet analysis
- **JSON** — For programmatic processing
- **PDF** — For compliance documentation

### How to Export

1. Go to **Audit Trail**
2. Apply filters for the data you need
3. Click **Export**
4. Select format
5. Download the file

## Retention

Audit logs are retained based on your plan:

| Plan | Retention |
|------|-----------|
| Standard | 90 days |
| Professional | 1 year |
| Enterprise | 7 years |

Longer retention is available for compliance requirements.

## Common Use Cases

### Compliance Audits

Prepare for audits by:
1. Filtering to the audit period
2. Exporting relevant events
3. Generating compliance reports

### Incident Investigation

Investigate security incidents:
1. Search for the affected user or resource
2. Review the timeline of events
3. Identify the root cause

### Access Reviews

Conduct periodic access reviews:
1. Filter by sensitive data access
2. Review who accessed what
3. Verify access was appropriate
