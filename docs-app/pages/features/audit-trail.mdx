# Audit Trail

Complete logging of all data access and governance actions.

## Overview

![Audit Trail](/screenshots/audit-trail.png)

SecureLens maintains a comprehensive audit trail:

- **Data Access**: Who accessed what data
- **Policy Actions**: Governance decisions
- **Configuration Changes**: System modifications
- **User Actions**: Administrative activities

## Audit Events

### Event Categories

| Category | Examples |
|----------|----------|
| **Data Access** | Query, export, view |
| **Classification** | Label assigned, changed |
| **Policy** | Evaluated, violated, enforced |
| **AI Gate** | Prompt sent, response filtered |
| **User** | Login, logout, permission change |
| **System** | Config change, scan completed |

### Event Structure

```json
{
  "event_id": "evt_abc123",
  "timestamp": "2024-01-15T10:30:00.000Z",
  "tenant_id": "tenant_xyz",
  "category": "data_access",
  "action": "query",
  "actor": {
    "type": "user",
    "id": "user_123",
    "email": "analyst@company.com",
    "ip_address": "192.168.1.100"
  },
  "resource": {
    "type": "table",
    "source": "prod-db",
    "name": "customers",
    "columns": ["email", "phone"]
  },
  "context": {
    "application": "analytics-dashboard",
    "session_id": "sess_456"
  },
  "result": {
    "status": "allowed",
    "policy_applied": "pii-access-policy",
    "rows_returned": 150
  }
}
```

## Viewing Audit Logs

### Search & Filter

Filter audit events by:

- **Time range**: Last hour, day, week, custom
- **Category**: Data access, policy, user, etc.
- **Actor**: User, service, system
- **Resource**: Data source, table, column
- **Result**: Allowed, denied, error

### Query Examples

Find all PII access:

```
category:data_access AND resource.pii_types:*
```

Find policy violations:

```
category:policy AND result.status:denied
```

Find user activity:

```
actor.email:analyst@company.com AND timestamp:[now-7d TO now]
```

## Data Access Auditing

### Query Logging

Every data query is logged:

| Field | Description |
|-------|-------------|
| **User** | Who ran the query |
| **Source** | Data source accessed |
| **Tables** | Tables queried |
| **Columns** | Columns accessed |
| **Filters** | WHERE conditions |
| **Rows** | Records returned |
| **Duration** | Query time |

### Sensitive Data Access

Enhanced logging for sensitive data:

- PII types accessed
- Sensitivity labels
- Policy evaluation
- Redaction applied

## Retention & Storage

### Retention Policy

Configure audit retention:

```yaml
audit:
  retention:
    default: 90d
    sensitive_access: 365d
    policy_violations: 730d
    
  archive:
    enabled: true
    destination: s3://audit-archive/
    after: 30d
```

### Storage Options

| Option | Use Case |
|--------|----------|
| **PostgreSQL** | Default, small-medium |
| **Elasticsearch** | Large scale, fast search |
| **S3 + Athena** | Long-term archive |
| **Splunk** | Enterprise SIEM |

## Export & Integration

### Export Formats

Export audit logs:

- JSON (default)
- CSV
- CEF (Common Event Format)
- LEEF (Log Event Extended Format)

### SIEM Integration

Stream to SIEM platforms:

```yaml
siem_integration:
  type: splunk
  endpoint: https://splunk.company.com:8088
  token: ${SPLUNK_HEC_TOKEN}
  index: securelens_audit
  
  # Or Elasticsearch
  type: elasticsearch
  endpoint: https://elastic.company.com:9200
  index_pattern: securelens-audit-%Y.%m.%d
```

### Webhook Integration

Send events to webhooks:

```yaml
webhooks:
  - name: "Security alerts"
    url: https://security.company.com/webhook
    events:
      - policy.violation
      - user.login_failed
    headers:
      Authorization: Bearer ${WEBHOOK_TOKEN}
```
