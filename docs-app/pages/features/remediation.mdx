# Remediation

Resolve data governance issues with automated workflows.


## Overview

SecureLens remediation helps you:

- **Identify issues** from scans and policies
- **Create remediation tasks** automatically
- **Track resolution** through workflows
- **Verify fixes** with re-scanning

## Issue Types

### Classification Issues

| Issue | Description | Remediation |
|-------|-------------|-------------|
| **Unclassified PII** | Sensitive data not labeled | Apply classification |
| **Misclassified** | Wrong classification | Correct classification |
| **Low confidence** | Uncertain classification | Manual review |

### Policy Violations

| Issue | Description | Remediation |
|-------|-------------|-------------|
| **Access violation** | Unauthorized access | Revoke access |
| **Retention violation** | Data past retention | Archive/delete |
| **Residency violation** | Data in wrong region | Move data |

### Data Quality Issues

| Issue | Description | Remediation |
|-------|-------------|-------------|
| **Missing values** | High null rate | Data enrichment |
| **Invalid format** | Format violations | Data cleansing |
| **Duplicates** | Redundant records | Deduplication |

### ROT Issues

| Issue | Description | Remediation |
|-------|-------------|-------------|
| **Obsolete data** | Past retention | Archive/delete |
| **Redundant data** | Duplicates | Consolidate |
| **Trivial data** | Low value | Delete |

## Remediation Workflow

### Workflow Stages

```
IDENTIFIED → TRIAGED → ASSIGNED → IN PROGRESS → RESOLVED → VERIFIED
     │          │          │           │            │          │
     ▼          ▼          ▼           ▼            ▼          ▼
  Detect    Prioritize  Assign to   Execute     Complete   Re-scan
  issue     and plan    owner       fix         task       to verify
```

```yaml

  type: POLICY_VIOLATION
  status: IN_PROGRESS
```

## Automated Remediation

### Auto-Remediation Rules

```yaml
auto_remediation:
  - name: Auto-Archive Obsolete Logs
    trigger:
      issue_type: ROT
      sub_type: OBSOLETE
      resource_pattern: "*_logs"
    action:
      type: ARCHIVE
      destination: s3://archive/logs/
    approval: false
    
  - name: Auto-Redact Exposed PII
    trigger:
      issue_type: POLICY_VIOLATION
      policy: PII_EXPOSURE
    action:
      type: REDACT
      method: MASK
    approval: true
    approvers: [data-steward@company.com]
```

### Remediation Actions

| Action | Description | Risk |
|--------|-------------|------|
| **REDACT** | Mask sensitive data | Low |
| **ENCRYPT** | Encrypt data at rest | Low |
| **ARCHIVE** | Move to cold storage | Low |
| **DELETE** | Permanently remove | High |
| **MOVE** | Relocate data | Medium |
| **REVOKE** | Remove access | Medium |
| **NOTIFY** | Alert stakeholders | Low |

## Manual Remediation

### Creating a Remediation Task

1. Navigate to **Remediation → Issues**
2. Select an issue
3. Click **Create Task**
4. Configure:

```yaml
task:
  title: Remediate SSN Exposure
  description: Remove SSN column from analytics view
  issue_id: iss-123
  assigned_to: analyst@company.com
  due_date: 2024-01-22
  priority: P1
  
  steps:
    - description: Identify affected views
      status: PENDING
    - description: Remove SSN from view definition
      status: PENDING
    - description: Verify removal
      status: PENDING
    - description: Update documentation
      status: PENDING
```

## Verification

### Re-Scan Verification

After remediation, SecureLens re-scans to verify:

```yaml
verification:
  issue_id: iss-123
  remediation_action: REDACT
  
  pre_remediation:
    pii_detected: true
    records_affected: 45230
    
  post_remediation:
    pii_detected: false
    records_affected: 0
    
  result: VERIFIED
  verified_at: "2024-01-21T15:00:00Z"
```

## API Access

### List Issues

```bash
curl -X GET "https://api.your-domain.com/v1/remediation/issues" \
  -H "Authorization: Bearer $API_KEY" \
  -d "status=OPEN&severity=HIGH"
```

### Create Task

```bash
curl -X POST "https://api.your-domain.com/v1/remediation/tasks" \
  -H "Authorization: Bearer $API_KEY" \
  -d '{
    "issue_id": "iss-123",
    "assigned_to": "analyst@company.com",
    "action": "REDACT"
  }'
```
