# Observability

Monitor, alert, and analyze SecureLens operations.

![Observability](/screenshots/observability.png)

## Overview

SecureLens provides comprehensive observability:

- **Metrics** — Prometheus-compatible metrics
- **Logs** — Structured JSON logging
- **Traces** — OpenTelemetry distributed tracing
- **Dashboards** — Pre-built Grafana dashboards
- **Alerts** — Configurable alerting rules

## Metrics

### Available Metrics

| Metric | Type | Description |
|--------|------|-------------|
| `securelens_requests_total` | Counter | Total API requests |
| `securelens_request_duration_seconds` | Histogram | Request latency |
| `securelens_classifications_total` | Counter | Classifications performed |
| `securelens_pii_detected_total` | Counter | PII entities detected |
| `securelens_policy_evaluations_total` | Counter | Policy evaluations |
| `securelens_policy_violations_total` | Counter | Policy violations |
| `securelens_scan_duration_seconds` | Histogram | Scan duration |
| `securelens_active_connections` | Gauge | Active DB connections |

### Prometheus Endpoint

```bash
# Scrape metrics
curl http://localhost:9090/metrics

# Sample output
securelens_requests_total{method="GET",path="/api/v1/classifications",status="200"} 12456
securelens_request_duration_seconds_bucket{le="0.1"} 10234
securelens_classifications_total{type="EMAIL"} 45230
securelens_pii_detected_total{type="SSN"} 1234
```

### Prometheus Configuration

```yaml
# prometheus.yml
scrape_configs:
  - job_name: 'securelens'
    static_configs:
      - targets: ['securelens-api:9090']
    metrics_path: /metrics
    scrape_interval: 15s
```

## Logging

### Log Format

```json
{
  "timestamp": "2024-01-20T10:30:00.000Z",
  "level": "INFO",
  "message": "Classification completed",
  "service": "securelens-api",
  "trace_id": "abc123",
  "span_id": "def456",
  "data_source_id": "ds-123",
  "records_classified": 1000,
  "pii_found": 45,
  "duration_ms": 1250
}
```

### Log Levels

| Level | Description | Use Case |
|-------|-------------|----------|
| `ERROR` | Errors requiring attention | Failures, exceptions |
| `WARN` | Potential issues | Degraded performance |
| `INFO` | Normal operations | Request completion |
| `DEBUG` | Detailed debugging | Development |
| `TRACE` | Very detailed | Deep debugging |

### Log Configuration

```yaml
logging:
  level: INFO
  format: json
  output:
    - stdout
    - file: /var/log/securelens/app.log
  rotation:
    max_size: 100MB
    max_age: 7 days
    max_backups: 5
```

## Alerting

### Alert Rules

```yaml
# alerting-rules.yml
groups:
  - name: securelens
    rules:
      - alert: HighErrorRate
        expr: rate(securelens_requests_total{status=~"5.."}[5m]) > 0.01
        for: 5m
        labels:
          severity: critical
        annotations:
          summary: High error rate detected
          
      - alert: SlowClassification
        expr: histogram_quantile(0.99, securelens_classification_duration_seconds) > 5
        for: 10m
        labels:
          severity: warning
        annotations:
          summary: Classification latency is high
          
      - alert: PolicyViolationSpike
        expr: rate(securelens_policy_violations_total[5m]) > 10
        for: 5m
        labels:
          severity: critical
        annotations:
          summary: Unusual policy violation rate
```

### Notification Channels

| Channel | Configuration |
|---------|---------------|
| **Email** | SMTP settings |
| **Slack** | Webhook URL |
| **PagerDuty** | Integration key |
| **Webhook** | Custom endpoint |

### Alert Configuration

```yaml
notifications:
  channels:
    - name: slack-alerts
      type: slack
      webhook_url: https://hooks.slack.com/...
      
    - name: pagerduty
      type: pagerduty
      integration_key: abc123
      
  routing:
    - match:
        severity: critical
      receivers: [pagerduty, slack-alerts]
      
    - match:
        severity: warning
      receivers: [slack-alerts]
```
