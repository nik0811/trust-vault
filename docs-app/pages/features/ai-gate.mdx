# AI Gate

Protect your LLM interactions with automatic scanning, redaction, and audit.

## What AI Gate Does

AI Gate is SecureLens's LLM proxy that sits between your applications and AI providers. It:

- **Scans prompts** for sensitive data before they reach the LLM
- **Redacts PII** automatically to prevent data exposure
- **Filters responses** for potential data leakage
- **Audits everything** for compliance and visibility
- **Tracks usage** including tokens and costs

## How It Works

```
Your App → AI Gate → LLM Provider
              ↓
         Scan & Redact
         Audit & Log
```

1. Your application sends a request to AI Gate
2. AI Gate scans the prompt for sensitive data
3. Detected PII is redacted based on your policies
4. The clean prompt is sent to the LLM provider
5. The response is scanned and filtered
6. The clean response is returned to your application
7. Everything is logged for audit

## Setting Up AI Gate

### Step 1: Configure Your Provider

1. Go to **AI Gate** → **Providers**
2. Click **Add Provider**
3. Select your LLM provider
4. Enter your API credentials
5. Click **Save**

### Step 2: Configure Scanning

1. Go to **AI Gate** → **Settings**
2. Enable **Scan Prompts** and **Scan Responses**
3. Select which PII types to detect
4. Set the confidence threshold

### Step 3: Configure Redaction

Choose how detected PII is handled:

| Method | Example |
|--------|---------|
| **Mask** | `John Smith` → `[PERSON]` |
| **Tokenize** | `John Smith` → `[TOKEN_123]` |
| **Remove** | `John Smith` → (removed) |

### Step 4: Update Your Application

Point your application to AI Gate instead of the LLM provider directly:

**Before:**
```
https://api.openai.com/v1/chat/completions
```

**After:**
```
https://api.securelens.ai/ai-gate/v1/chat/completions
```

Use your SecureLens API key for authentication.

## Example: Prompt Scanning

**Original prompt:**
```
Summarize this customer complaint:
Customer John Smith (SSN: 123-45-6789) called about 
their credit card 4111-1111-1111-1111 being charged twice.
```

**After redaction:**
```
Summarize this customer complaint:
Customer [PERSON] (SSN: [SSN REDACTED]) called about 
their credit card [CARD REDACTED] being charged twice.
```

The LLM receives the redacted version, protecting sensitive data.

## AI Governance Policies

Create policies specific to AI usage:

### Model Restrictions

Limit which models can be used:
- Only allow approved models (e.g., GPT-4, Claude 3)
- Block unapproved or experimental models

### Usage Limits

Set limits on AI usage:
- Daily token limits per user or team
- Request rate limits
- Cost caps

### Content Policies

Control what can be sent to AI:
- Block certain topics or keywords
- Require approval for specific use cases
- Restrict code generation to engineering teams

## Audit Trail

Every AI Gate request is logged:

- **Timestamp** — When the request was made
- **User** — Who made the request
- **Model** — Which LLM was used
- **Tokens** — Input and output token counts
- **PII Detected** — What sensitive data was found
- **Redactions** — What was redacted
- **Cost** — Estimated cost
