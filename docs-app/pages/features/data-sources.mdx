## Adding a Data Source

### Step 1: Navigate to Data Sources

1. Click **Data Sources** in the left sidebar
2. Click **Add Data Source**

### Step 2: Select Your Source Type

Choose the type of data source you want to connect. Options are organized by category.

### Step 3: Enter Connection Details

Fill in the required fields for your source type. For example, for a PostgreSQL database:

| Field | Description |
|-------|-------------|
| **Name** | A friendly name for this source |
| **Host** | Database server address |
| **Port** | Database port (default: 5432) |
| **Database** | Database name |
| **Username** | Database user |
| **Password** | Database password |
| **SSL Mode** | Connection security setting |

### Step 4: Test the Connection

Click **Test Connection** to verify:

- Network connectivity
- Authentication credentials
- Required permissions

### Step 5: Configure Scan Settings

| Setting | Description | Recommended |
|---------|-------------|-------------|
| **Scan Schedule** | How often to scan | Daily |
| **Sample Size** | Records to sample per table | 1,000 |
| **Include/Exclude** | Tables to include or skip | Include all |
| **Auto-classify** | Automatically classify data | Enabled |

### Step 6: Save and Start Scanning

Click **Save & Scan** to save the configuration and start the initial scan.

## Managing Data Sources

### View Data Source Details

1. Click on a data source name
2. View connection status, last scan time, and classification summary
3. See tables and columns discovered

### Edit Configuration

1. Click on a data source
2. Click **Edit**
3. Modify settings
4. Click **Save**

### Trigger a Manual Scan

1. Click on a data source
2. Click **Rescan**
3. Choose scan type:
   - **Full** — Rescan everything
   - **Incremental** — Only new/changed data
   - **Schema Only** — Just structure

### Disable or Enable

Temporarily disable a data source:

1. Click on a data source
2. Toggle **Enabled** off
3. Scans and policies pause until re-enabled

### Delete a Data Source

1. Click on a data source
2. Click **Delete**
3. Confirm deletion

> **Warning:** Deleting a data source removes all associated classifications and audit history.

## Troubleshooting

### Connection Failed

Check these common issues:

1. **Firewall** — Ensure SecureLens IP addresses are allowed
2. **Credentials** — Verify username and password
3. **Host/Port** — Confirm the server address is correct
4. **SSL** — Try different SSL modes

### Permission Denied

The database user may lack required permissions:

1. Verify SELECT access to tables
2. Check schema-level permissions
3. Ensure database access is granted

### Scan Timeout

The scan is taking too long:

1. Reduce sample size
2. Exclude large tables
3. Check database performance

### Missing Tables

Tables aren't appearing:

1. Verify schema permissions
2. Check include/exclude filters
3. Refresh the schema cache
