# Classification

AI-powered sensitive data detection and classification.

![Classification](/screenshots/classification.png)

## What Classification Does

SecureLens uses advanced AI to automatically detect and classify sensitive data in your connected sources. When you scan a data source, the classification engine:

1. **Samples data** from each table
2. **Analyzes content** using AI models
3. **Identifies sensitive data** like PII, PHI, and PCI
4. **Assigns confidence scores** to each detection
5. **Applies sensitivity labels** based on the findings

## How It Works

### Data Sampling

When a data source is scanned:
- SecureLens samples records from each table
- Default: 1,000 records per table
- Configurable per data source

### AI Classification

The classification engine analyzes text content and identifies entities:

**Example:**
```
Input: "Contact John Smith at john.smith@company.com or 555-123-4567"

Detected:
  - "John Smith" → PERSON_NAME (95% confidence)
  - "john.smith@company.com" → EMAIL_ADDRESS (99% confidence)
  - "555-123-4567" → PHONE_NUMBER (97% confidence)
```

### Aggregation

Classifications are aggregated at the column level:

| Column | PII Type | Confidence | Count |
|--------|----------|------------|-------|
| `customer_name` | PERSON_NAME | 98% | 950/1000 |
| `email` | EMAIL_ADDRESS | 99% | 1000/1000 |
| `phone` | PHONE_NUMBER | 95% | 875/1000 |

### Label Assignment

Based on classifications, sensitivity labels are assigned:

- **Highly Sensitive** — SSN, credit cards, health data
- **Sensitive** — Names, emails, addresses
- **Internal** — Business data, IDs
- **Public** — Non-sensitive data

## Viewing Classifications

### By Data Source

1. Go to **Classification**
2. Select **By Data Source**
3. Expand a source to see tables and columns
4. Click a column for details

### By PII Type

1. Go to **Classification**
2. Select **By PII Type**
3. Click a type (e.g., EMAIL) to see all occurrences

### By Sensitivity

1. Go to **Classification**
2. Select **By Sensitivity**
3. Filter by label (Highly Sensitive, Sensitive, etc.)

## Reviewing and Editing Classifications

### Confirm a Classification

When you verify a classification is correct:

1. Click on a classified column
2. Review the detected PII type and confidence
3. Click **Confirm**

### Correct a Classification

If the AI made a mistake:

1. Click on a classified column
2. Click **Edit Classification**
3. Select the correct PII type
4. Add a note explaining the correction
5. Click **Save**

Your feedback helps improve future classifications.

### Bulk Operations

For efficient review:

1. Go to **Classification**
2. Filter by confidence (e.g., 70-89%)
3. Select multiple items
4. Click **Bulk Confirm** or **Bulk Edit**

## Custom Entity Types

Define custom PII types for your domain:

### Create a Custom Type

1. Go to **Settings** → **Classification**
2. Click **Add Custom Type**
3. Configure:
   - **Name** — Internal identifier (e.g., EMPLOYEE_ID)
   - **Display Name** — Human-readable name
   - **Patterns** — Regex patterns to match
   - **Sensitivity** — Default sensitivity level

### Example Custom Types

- **Employee ID** — Internal employee identifiers
- **Project Code** — Project tracking codes
- **Customer Number** — Your customer ID format

## Confidence Thresholds

Configure how classifications are handled based on confidence:

| Threshold | Default | Behavior |
|-----------|---------|----------|
| **Auto-label** | 90% | Automatically apply labels |
| **Review** | 70% | Flag for human review |
| **Ignore** | 50% | Don't label |

Adjust these in **Settings** → **Classification**.
