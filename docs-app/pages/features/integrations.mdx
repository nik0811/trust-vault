## Configuring Integrations

### Slack Integration

1. Navigate to **Settings → Integrations**
2. Click **Slack**
3. Click **Connect to Slack**
4. Authorize SecureLens
5. Select channels for notifications

```yaml
slack:
  enabled: true
  webhook_url: https://hooks.slack.com/services/...
  channels:
    alerts: "#securelens-alerts"
    reports: "#compliance-reports"
  notifications:
    - type: POLICY_VIOLATION
      channel: alerts
      severity: [HIGH, CRITICAL]
    - type: SCAN_COMPLETE
      channel: reports
```

### Jira Integration

```yaml
jira:
  enabled: true
  url: https://company.atlassian.net
  username: securelens@company.com
  api_token: ${JIRA_API_TOKEN}
  project: SEC
  issue_types:
    policy_violation: Bug
    remediation: Task
  custom_fields:
    severity: customfield_10001
    data_source: customfield_10002
```

### Splunk Integration

```yaml
splunk:
  enabled: true
  hec_url: https://splunk.company.com:8088
  hec_token: ${SPLUNK_HEC_TOKEN}
  index: securelens
  source: securelens-api
  events:
    - AUDIT_LOG
    - POLICY_VIOLATION
    - CLASSIFICATION
```

## Webhooks

### Outbound Webhooks

Send events to any HTTP endpoint:

```yaml
webhooks:
  - name: custom-siem
    url: https://siem.company.com/api/events
    method: POST
    headers:
      Authorization: Bearer ${SIEM_TOKEN}
    events:
      - POLICY_VIOLATION
      - DATA_ACCESS
    retry:
      max_attempts: 3
      backoff: exponential
```

### Webhook Payload

```json
{
  "event_type": "POLICY_VIOLATION",
  "timestamp": "2024-01-20T10:30:00Z",
  "data": {
    "policy_id": "pol-123",
    "policy_name": "PII Access Control",
    "user": "analyst@company.com",
    "resource": "users.ssn",
    "action": "DENIED",
    "reason": "User lacks required role"
  },
  "metadata": {
    "tenant_id": "tenant-abc",
    "source": "securelens"
  }
}
```

### Inbound Webhooks

Receive events from external systems:

```yaml
inbound_webhooks:
  - name: github-events
    path: /webhooks/github
    secret: ${GITHUB_WEBHOOK_SECRET}
    actions:
      push:
        - trigger_scan: true
```

## API Integration

### REST API

Full REST API for custom integrations:

```bash
# Authentication
curl -X POST "https://api.securelens.ai/v1/auth/token" \
  -d "client_id=xxx&client_secret=yyy"

# Get classifications
curl -X GET "https://api.securelens.ai/v1/classifications" \
  -H "Authorization: Bearer $TOKEN"
```

### GraphQL API

```graphql
query GetClassifications {
  classifications(dataSourceId: "ds-123") {
    id
    column
    type
    confidence
    createdAt
  }
}
```

## Data Export

### Export Destinations

| Destination | Format | Schedule |
|-------------|--------|----------|
| **S3** | JSON, Parquet | Hourly/Daily |
| **Azure Blob** | JSON, Parquet | Hourly/Daily |
| **GCS** | JSON, Parquet | Hourly/Daily |
| **SFTP** | CSV, JSON | Daily |
| **Snowflake** | Direct load | Real-time |

### Export Configuration

```yaml
exports:
  - name: audit-to-s3
    destination:
      type: s3
      bucket: securelens-exports
      prefix: audit/
      region: us-east-1
    data:
      - AUDIT_LOGS
      - CLASSIFICATIONS
    format: parquet
    schedule: "0 * * * *"  # Hourly
    partition_by: [date, tenant_id]
```
