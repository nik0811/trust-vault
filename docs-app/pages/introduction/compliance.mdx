# Compliance Frameworks

SecureLens provides built-in support for major regulatory frameworks and industry standards.

## Supported Frameworks

### GDPR (General Data Protection Regulation)

The EU's comprehensive data protection regulation.

| Requirement | How SecureLens Helps |
|-------------|----------------------|
| **Article 30**: Records of processing | Automated data mapping across all sources |
| **Article 32**: Security of processing | Access controls and encryption monitoring |
| **Article 33**: Breach notification | Real-time detection and alerting |
| **Article 35**: DPIA | Risk assessment reports |
| **Article 17**: Right to erasure | Data discovery and deletion tracking |

**Key Features:**
- Automated PII detection across all data sources
- Consent management integration
- Data subject request automation
- Cross-border transfer monitoring

### CCPA/CPRA (California Consumer Privacy Act)

California's landmark privacy legislation.

| Requirement | How SecureLens Helps |
|-------------|----------------------|
| **Right to Know** | Data inventory and access logs |
| **Right to Delete** | Data discovery and deletion verification |
| **Right to Opt-Out** | Consent tracking and policy enforcement |
| **Data Minimization** | ROT detection and retention policies |

**Key Features:**
- Consumer data inventory
- Sale/sharing tracking
- Automated opt-out enforcement
- Annual reporting support

### HIPAA (Health Insurance Portability and Accountability Act)

US healthcare data protection standard.

| Requirement | How SecureLens Helps |
|-------------|----------------------|
| **Privacy Rule** | PHI detection and access controls |
| **Security Rule** | Encryption and audit trails |
| **Breach Notification** | Real-time detection and reporting |
| **Minimum Necessary** | Policy-based data access |

**Key Features:**
- 18 HIPAA identifiers detection
- BAA compliance tracking
- ePHI access monitoring
- Breach risk assessment

### PCI-DSS (Payment Card Industry Data Security Standard)

Credit card data protection requirements.

| Requirement | How SecureLens Helps |
|-------------|----------------------|
| **Req 3**: Protect stored data | PAN detection and encryption monitoring |
| **Req 7**: Restrict access | RBAC and least privilege enforcement |
| **Req 10**: Track access | Comprehensive audit logs |
| **Req 12**: Security policies | Policy enforcement and monitoring |

**Key Features:**
- Credit card number detection
- Tokenization support
- Access logging
- Quarterly scan reports

### SOX (Sarbanes-Oxley Act)

Financial reporting and controls.

| Requirement | How SecureLens Helps |
|-------------|----------------------|
| **Section 302**: Disclosure controls | Data classification |
| **Section 404**: Internal controls | Access controls and audit |
| **Section 409**: Real-time disclosure | Anomaly detection |

### SOC 2 (Service Organization Control)

Trust service criteria for service providers.

| Trust Criteria | How SecureLens Helps |
|----------------|----------------------|
| **Security** | Access controls and encryption |
| **Availability** | Monitoring and alerting |
| **Processing Integrity** | Data quality and validation |
| **Confidentiality** | Classification and redaction |
| **Privacy** | PII protection and consent |

## Compliance Dashboard

The SecureLens dashboard provides a unified view of your compliance posture:

- **Framework scores** — See your compliance percentage for each framework
- **Control status** — Track which controls are passing or need attention
- **Upcoming audits** — Stay on top of audit deadlines
- **Remediation tasks** — Prioritized list of issues to address

## Automated Reporting

Generate compliance reports with one click:

- **Data Inventory Report** — All data sources and classifications
- **Access Audit Report** — Who accessed what, when
- **Policy Compliance Report** — Policy violations and remediation
- **Risk Assessment Report** — Data risk scores and trends
- **Breach Readiness Report** — Incident response preparedness
