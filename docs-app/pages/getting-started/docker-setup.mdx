## Quick Start

### 1. Clone Repository

```bash
git clone https://github.com/securelens/securelens.git
cd securelens
```

### 2. Create Environment File

```bash
cp .env.example .env
```

### 3. Start All Services

```bash
docker-compose up -d
```

### 4. Check Status

```bash
docker-compose ps
docker-compose logs -f
```

## Docker Compose File

Here's the complete `docker-compose.yml`:

```yaml
version: '3.8'

services:
  # Frontend UI
  ui:
    image: securelens/ui:latest
    ports:
      - "3000:3000"
    environment:
      - NEXT_PUBLIC_API_URL=http://api:8080
    depends_on:
      - api
    restart: unless-stopped

  # Backend API
  api:
    image: securelens/api:latest
    ports:
      - "8080:8080"
    environment:
      - DATABASE_URL=postgres://securelens:securelens@postgres:5432/securelens
      - KAFKA_BROKERS=kafka:9092
      - QDRANT_URL=http://qdrant:6333
      - JWT_SECRET=${JWT_SECRET}
      - ADMIN_EMAIL=${ADMIN_EMAIL}
      - ADMIN_PASSWORD=${ADMIN_PASSWORD}
    depends_on:
      postgres:
        condition: service_healthy
      kafka:
        condition: service_healthy
    restart: unless-stopped

  # Background Worker
  worker:
    image: securelens/api:latest
    command: ["--mode=worker"]
    environment:
      - DATABASE_URL=postgres://securelens:securelens@postgres:5432/securelens
      - KAFKA_BROKERS=kafka:9092
      - QDRANT_URL=http://qdrant:6333
    depends_on:
      - api
    restart: unless-stopped

  # PostgreSQL Database
  postgres:
    image: postgres:15-alpine
    environment:
      - POSTGRES_USER=securelens
      - POSTGRES_PASSWORD=securelens
      - POSTGRES_DB=securelens
    volumes:
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U securelens"]
      interval: 5s
      timeout: 5s
      retries: 5
    restart: unless-stopped

  # Apache Kafka (KRaft mode)
  kafka:
    image: bitnami/kafka:3.6
    environment:
      - KAFKA_CFG_NODE_ID=0
      - KAFKA_CFG_PROCESS_ROLES=controller,broker
      - KAFKA_CFG_LISTENERS=PLAINTEXT://:9092,CONTROLLER://:9093
      - KAFKA_CFG_LISTENER_SECURITY_PROTOCOL_MAP=CONTROLLER:PLAINTEXT,PLAINTEXT:PLAINTEXT
      - KAFKA_CFG_CONTROLLER_QUORUM_VOTERS=0@kafka:9093
      - KAFKA_CFG_CONTROLLER_LISTENER_NAMES=CONTROLLER
    volumes:
      - kafka_data:/bitnami/kafka
    healthcheck:
      test: ["CMD-SHELL", "kafka-topics.sh --bootstrap-server localhost:9092 --list"]
      interval: 10s
      timeout: 10s
      retries: 5
    restart: unless-stopped

  # Qdrant Vector Database
  qdrant:
    image: qdrant/qdrant:v1.8.0
    volumes:
      - qdrant_data:/qdrant/storage
    restart: unless-stopped

volumes:
  postgres_data:
  kafka_data:
  qdrant_data:
```

## Environment Variables

### Required Variables

| Variable | Description | Example |
|----------|-------------|---------|
| `JWT_SECRET` | Secret for JWT signing | `your-256-bit-secret` |
| `ADMIN_EMAIL` | Initial admin email | `admin@company.com` |
| `ADMIN_PASSWORD` | Initial admin password | `SecurePass123!` |

### Optional Variables

| Variable | Description | Default |
|----------|-------------|---------|
| `DATABASE_URL` | PostgreSQL connection | `postgres://...` |
| `KAFKA_BROKERS` | Kafka broker list | `kafka:9092` |
| `QDRANT_URL` | Qdrant endpoint | `http://qdrant:6333` |
| `LOG_LEVEL` | Logging verbosity | `info` |
| `OPENAI_API_KEY` | OpenAI API key | - |

## Production Configuration

### Enable TLS

```yaml
services:
  ui:
    labels:
      - "traefik.enable=true"
      - "traefik.http.routers.ui.rule=Host(`securelens.company.com`)"
      - "traefik.http.routers.ui.tls=true"
      - "traefik.http.routers.ui.tls.certresolver=letsencrypt"
```

### Resource Limits

```yaml
services:
  api:
    deploy:
      resources:
        limits:
          cpus: '4'
          memory: 8G
        reservations:
          cpus: '2'
          memory: 4G
```

### External Database

```yaml
services:
  api:
    environment:
      - DATABASE_URL=postgres://user:pass@rds.amazonaws.com:5432/securelens?sslmode=require
```

### Scaling Workers

```bash
docker-compose up -d --scale worker=4
```

## Health Checks

### Check All Services

```bash
# Service status
docker-compose ps

# API health
curl http://localhost:8080/health

# UI health
curl http://localhost:3000/api/health
```

### View Logs

```bash
# All services
docker-compose logs -f

# Specific service
docker-compose logs -f api

# Last 100 lines
docker-compose logs --tail=100 api
```

## Backup and Restore

### Backup PostgreSQL

```bash
docker-compose exec postgres pg_dump -U securelens securelens > backup.sql
```

### Restore PostgreSQL

```bash
cat backup.sql | docker-compose exec -T postgres psql -U securelens securelens
```

### Backup Volumes

```bash
docker run --rm -v securelens_postgres_data:/data -v $(pwd):/backup \
  alpine tar czf /backup/postgres_backup.tar.gz /data
```

## Upgrading

### Pull Latest Images

```bash
docker-compose pull
```

### Apply Upgrade

```bash
docker-compose up -d
```

### Run Migrations

```bash
docker-compose exec api securelens migrate up
```

## Troubleshooting

### Container Won't Start

```bash
# Check logs
docker-compose logs api

# Check resources
docker system df
docker system prune -a
```

### Database Connection Issues

```bash
# Test connection
docker-compose exec api pg_isready -h postgres -U securelens

# Check PostgreSQL logs
docker-compose logs postgres
```

### Kafka Issues

```bash
# List topics
docker-compose exec kafka kafka-topics.sh --bootstrap-server localhost:9092 --list

# Check consumer lag
docker-compose exec kafka kafka-consumer-groups.sh --bootstrap-server localhost:9092 --describe --all-groups
```
