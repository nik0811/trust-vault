# First Login

Guide to your first login and initial setup of SecureLens.

## Accessing the UI

After starting SecureLens, open your browser and navigate to:

```
http://localhost:3000
```

For production deployments, use your configured domain:
```
https://securelens.yourcompany.com
```

## Login Screen

You'll see the SecureLens login page:

![Login Screen](/screenshots/login.png)

Enter your credentials:
- **Email**: The admin email configured in `.env`
- **Password**: The admin password configured in `.env`

Default credentials (if not changed):
- Email: `admin@securelens.local`
- Password: `SecureLens@2026!`

> **Warning:** Change the default password immediately after first login!

## Initial Setup Wizard

On first login, you'll be guided through the setup wizard:

### Step 1: Organization Profile

Configure your organization details:

- **Organization Name**: Your company name
- **Industry**: Select your industry (affects compliance recommendations)
- **Size**: Number of employees
- **Primary Region**: Data residency region

### Step 2: Admin Account

Secure your admin account:

1. Change the default password
2. Enable two-factor authentication (recommended)
3. Add a recovery email

### Step 3: Connect First Data Source

Add your first data source to start discovering data:

1. Click **Add Data Source**
2. Select the data source type:
   - PostgreSQL
   - MySQL
   - SQL Server
   - MongoDB
   - S3/GCS/Azure Blob
   - Snowflake
   - BigQuery

3. Enter connection details
4. Test the connection
5. Start the initial scan

### Step 4: Review Classification Results

After the scan completes:

1. Review discovered tables and columns
2. Verify automatic classifications
3. Add custom labels if needed
4. Approve or correct classifications

## Quick Actions

### Add More Data Sources

1. Navigate to **Data Sources**
2. Click **Add Data Source**
3. Follow the connection wizard

### Create a Policy

1. Navigate to **Governance** → **Policies**
2. Click **Create Policy**
3. Define conditions and actions
4. Activate the policy

### Invite Team Members

1. Navigate to **Settings** → **Users**
2. Click **Invite User**
3. Enter email and select role
4. Send invitation

## User Roles

SecureLens supports role-based access control:

| Role | Permissions |
|------|-------------|
| **Admin** | Full access to all features |
| **Analyst** | View data, manage classifications |
| **Auditor** | Read-only access to audit logs |
| **Developer** | API access, limited UI access |

## Troubleshooting

### Can't Login

1. Verify credentials are correct
2. Check if the API is running: `curl http://localhost:8080/health`
3. Clear browser cache and cookies
4. Check API logs: `docker-compose logs api`

### Forgot Password

1. Click "Forgot Password" on login page
2. Enter your email
3. Check for reset email
4. Follow the reset link

### Account Locked

After 5 failed login attempts, accounts are locked for 15 minutes.

To unlock immediately (admin):
```bash
docker-compose exec api securelens user unlock admin@company.com
```

## Security Best Practices

1. **Change default credentials** immediately
2. **Enable 2FA** for all admin accounts
3. **Use SSO** for enterprise deployments
4. **Review audit logs** regularly
5. **Rotate API keys** periodically
