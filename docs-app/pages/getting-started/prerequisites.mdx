# Prerequisites

Detailed requirements for running SecureLens in different environments.

## System Requirements

### Minimum Requirements

| Resource | Minimum | Recommended |
|----------|---------|-------------|
| **CPU** | 4 cores | 8+ cores |
| **RAM** | 8 GB | 16+ GB |
| **Disk** | 20 GB SSD | 100+ GB SSD |
| **Network** | 100 Mbps | 1 Gbps |

### Production Requirements

For production deployments processing millions of records:

| Resource | Small | Medium | Large |
|----------|-------|--------|-------|
| **CPU** | 8 cores | 16 cores | 32+ cores |
| **RAM** | 32 GB | 64 GB | 128+ GB |
| **Disk** | 500 GB | 1 TB | 2+ TB |
| **Kafka Partitions** | 12 | 24 | 48+ |

## Software Requirements

### Docker (Recommended)

```bash
# Check Docker version
docker --version
# Required: 24.0+

# Check Docker Compose version
docker compose version
# Required: v2.20+
```

### Kubernetes (Production)

For Kubernetes deployments:

- Kubernetes 1.27+
- Helm 3.12+
- Ingress controller (nginx, traefik, etc.)
- Storage class with ReadWriteOnce support
- Optional: Cert-manager for TLS

### Standalone (Development)

For running without containers:

- Go 1.22+
- Node.js 20+
- PostgreSQL 15+
- Apache Kafka 3.6+ (KRaft mode)
- Qdrant 1.8+

## Network Requirements

### Ports

| Port | Service | Protocol | Direction |
|------|---------|----------|-----------|
| 3000 | UI | HTTP/HTTPS | Inbound |
| 8080 | API | HTTP/HTTPS | Inbound |
| 5432 | PostgreSQL | TCP | Internal |
| 9092 | Kafka | TCP | Internal |
| 6333 | Qdrant | HTTP | Internal |
| 6334 | Qdrant gRPC | gRPC | Internal |

### Firewall Rules

For cloud deployments, allow:

```bash
# Inbound (from users/clients)
TCP 443 (HTTPS)
TCP 80 (HTTP redirect)

# Internal (between services)
TCP 5432 (PostgreSQL)
TCP 9092 (Kafka)
TCP 6333-6334 (Qdrant)
TCP 8080 (API internal)
```

### DNS

- Public DNS for UI access
- Internal DNS for service discovery (Kubernetes)
- Optional: Split-horizon DNS for hybrid deployments

## External Services

### Required

| Service | Purpose | Alternatives |
|---------|---------|--------------|
| **PostgreSQL** | Primary database | AWS RDS, Azure Database, Cloud SQL |
| **Kafka** | Event streaming | AWS MSK, Confluent Cloud, Redpanda |

### Optional

| Service | Purpose | When Needed |
|---------|---------|-------------|
| **Qdrant** | Vector search | AI Gate, semantic search |
| **Redis** | Caching | High-traffic deployments |
| **S3/GCS/Azure Blob** | Document storage | Document processing |
| **SMTP** | Email notifications | Alerts, reports |
| **OpenAI/Azure OpenAI** | LLM integration | AI Gate features |

## Cloud Provider Requirements

### AWS

```yaml
# Minimum IAM permissions
- ec2:*  # For compute
- rds:*  # For PostgreSQL
- msk:*  # For Kafka
- s3:*   # For document storage
- secretsmanager:GetSecretValue  # For secrets
```

### Azure

```yaml
# Required resource providers
- Microsoft.Compute
- Microsoft.DBforPostgreSQL
- Microsoft.EventHub  # Kafka-compatible
- Microsoft.Storage
- Microsoft.KeyVault
```

### GCP

```yaml
# Required APIs
- compute.googleapis.com
- sqladmin.googleapis.com
- pubsub.googleapis.com  # Or Confluent for Kafka
- storage.googleapis.com
- secretmanager.googleapis.com
```

## Security Requirements

### TLS/SSL

- TLS 1.2+ required for all external connections
- Valid SSL certificates for production
- Let's Encrypt supported via cert-manager

### Authentication

- OIDC provider (Okta, Auth0, Azure AD) for SSO
- Or built-in JWT authentication
- API keys for service-to-service

### Secrets Management

Supported secret stores:
- Kubernetes Secrets
- HashiCorp Vault
- AWS Secrets Manager
- Azure Key Vault
- GCP Secret Manager

## Verification Checklist

Before installation, verify:

```bash
# Docker
docker --version  # 24.0+
docker compose version  # v2.20+

# Resources
docker system info | grep -E "Memory|CPUs"
# Memory: 8GB+, CPUs: 4+

# Disk space
df -h /var/lib/docker
# 20GB+ available

# Ports
netstat -tlnp | grep -E "3000|8080|5432|9092"
# Should be empty (ports available)

# Network
curl -I https://registry.hub.docker.com
# Should return 200 OK
```
